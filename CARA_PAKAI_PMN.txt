--------------------------------------------------------------------
1. ARSITEKTUR KARYA: HYBRID MODULAR-MONOLITH (MENGAPA BEGINI?)
--------------------------------------------------------------------
Untuk efisiensi token AI dan kecepatan kompilasi, proyek ini dibagi:
*   KOTAK EDIT NASKAH (DATABASE CHUNKS): Folder `data/parts/` berisi 21 
    berkas JSON bab berukuran sangat kecil (~10KB s.d ~300KB).
*   KOTAK GLOSARIUM KANONIK: File `data/gl.json` (kamus istilah) dan 
    `data/glg.json` (kategori istilah).
*   KOTAK EDIT TAMPILAN (UI): File `index.ui.html` (~40KB).
*   KOTAK DESAIN & LOGIKA: Berkas `style.css` dan `app.js`.
*   HASIL AKHIR PRODUKSI (MONOLITH RAKSASA): Berkas tunggal `index.html` 
    (~2.58 MB) yang digabungkan otomatis oleh skrip compiler.

Aturan Emas: JANGAN PERNAH mengedit file index.html secara langsung. 
Semua suntingan dilakukan pada berkas modular di atas, lalu dikompilasi.

--------------------------------------------------------------------
2. PUSAT KENDALI UTAMA (SATU PINTU): 00_PMN_WORKSPACE.bat
--------------------------------------------------------------------
Cukup klik ganda (double-click) file sakti di folder utama Anda:
   D:\pmn-framework\00_PMN_WORKSPACE.bat

Konsol interaktif berwarna akan muncul di layar PC Anda. Cukup ketik
angka pilihan Anda [0 s.d 10] lalu tekan Enter:

*   Tekan [1] : LIGHTNING COMPILE (Kompilasi Visual + Data + Corpus)
    --> Menggabungkan semua berkas modular, gaya CSS, logika JS, dan
        glosarium menjadi file produksi `index.html` tunggal dalam 1 detik.
        Dilengkapi Checklist Pengaman berlapis untuk mencegah rilis kosong.
*   Tekan [2] : LAUNCH DEV SERVER (Server Pratinjau Lokal)
    --> Menjalankan server HTTP ringan pada http://localhost:8000 dan 
        otomatis membuka browser agar Anda bisa membaca perubahan secara realtime.
*   Tekan [3] : IMPORT WORD NASKAH (Impor Selektif & Kompilasi Otomatis)
    --> Memindai seluruh berkas Word (.docx) di folder `docx_source/`,
        menampilkan pilihan file, dan otomatis menguraikannya menjadi
        21 file JSON modular serta memperbarui versi di UI secara otomatis.
*   Tekan [4] : SPLIT MONOLITH TO JSON (Pecah Database Monolitik)
    --> Tindakan darurat untuk memotong paksa data/parts.json menjadi
        21 file JSON modular di folder data/parts/.
*   Tekan [5] : RESTORE STABLE BACKUP (Sistem Rollback Rilis Cepat)
    --> Mengembalikan file `index.html` aktif ke versi cadangan stabil 
        sebelumnya (`index.html.bak`) jika terjadi kegagalan visual mendadak.
*   Tekan [6] : SYSTEM BLUEPRINT (Cetak Biru Arsitektur AI)
    --> Membuka dan menampilkan rangkuman aturan AI.md langsung di layar.
*   Tekan [7] : SEARCH WORKSPACE (Pencarian Kode Sumber Cepat)
    --> Mencari kata kunci tertentu di seluruh folder proyek secara aman 
        dan super hemat token (membatasi penampilan baris panjang).
*   Tekan [8] : SECURE META REMOVER (Pembersih Metadata & Backup Telegram)
    --> Mensterilkan dokumen draf di folder raw_inputs/ dari nama asli Anda 
        ("Ali Ikhsan") sebelum didistribusikan secara publik.
*   Tekan [9] : SYSTEM DIAGNOSTICS (Deteksi Masalah & AI Rescue Packet)
    --> [FITUR BARU] Menjalankan pemindaian mandiri secara mendalam untuk 
        mendeteksi error sintaksis JSON, link mati, dan menghasilkan berkas
        bantuan khusus untuk AI web gratisan.
*   Tekan [10] : BACKUP & ROLLBACK (Ringan & Komplet berbasis Snapshot ZIP)
    --> [FITUR BARU] Membuat pencadangan arsip ZIP instan (hanya ~770 KB) 
        dari seluruh file kerja Anda atau memulihkan (rollback) seluruh workspace
        modular Anda ke titik waktu tertentu dengan aman.

LANGKAH 1: EDIT NASKAH DI MS WORD
   1. Buka berkas Word manuskrip Anda di Microsoft Word seperti biasa.
   2. Lakukan penulisan bab, koreksi kata, atau revisi sesuka hati Anda.
   3. Simpan berkas dengan format nama versi baru di folder PC Anda, 
      contoh: `PMN_Framework_v117.docx`.
   4. [PENTING!] Tutup Microsoft Word Anda agar berkas tidak terkunci!

LANGKAH 2: MASUKKAN KE FOLDER SUMBER
   1. Pindahkan berkas Word `.docx` baru Anda ke folder khusus ini:
      D:\pmn-framework\docx_source\
   2. Pastikan folder tersebut hanya berisi berkas `.docx` yang aktif Anda kerjakan.

LANGKAH 3: IMPOR & COMPILE SATU PINTU
   1. Klik ganda file: D:\pmn-framework\00_PMN_WORKSPACE.bat
   2. Tekan tombol [3] lalu Enter untuk mengimpor naskah Word Anda.
   3. Sistem secara otomatis mendeteksi versi (misal v117), mengurai XML, 
      memecah bab menjadi berkas JSON modular, memperbarui label versi di 
      template web, dan merajutnya langsung menjadi `index.html` siap saji!

LANGKAH 4: STERILKAN METADATA PRIBADI (PENTING!)
   1. Di konsol interaktif yang sama, tekan tombol [8] lalu Enter.
   2. Skrip otomatis menghapus seluruh metadata pribadi ("Ali Ikhsan") 
      pada dokumen draf di folder masukan, mengaudit tingkat kebocoran, 
      dan mengunggah cadangan sterilnya ke Telegram Log Group Anda!
   3. [Alternatif Drag-and-Drop]: Anda juga bisa langsung menyeret file 
      Word/PDF mentah Anda dan menjatuhkannya di atas file batch:
      D:\pmn-framework\BERSIHKAN_DAN_BACKUP_DOKUMEN.bat

LANGKAH 5: PUSH KE GITHUB SEKALIGUS (SATU KLIK)
   1. Cari file peluncur push otomatis di root folder Anda:
      D:\pmn-framework\KIRIM_KE_GITHUB.bat
   2. Klik ganda berkas `KIRIM_KE_GITHUB.bat` tersebut.
   3. Masukkan pesan pembaruan singkat (contoh: "Update Bab 8") lalu Enter.
   4. Perubahan terbaru Anda beserta draf PDF/DOCX steril telah resmi terbit 
      di GitHub publik tanpa meninggalkan jejak identitas pribadi!

4. SISTEM DIAGNOSTIK MANDIRI & PENYELAMATAN AI WEB (OFFLINE FOREVER)
--------------------------------------------------------------------
Jika di masa depan Anda sudah tidak berlangganan AI tingkat tinggi (seperti
Antigravity, Claude Code, dll.) dan mengalami kegagalan kompilasi (misalnya 
karena salah memasukkan karakter koma di file JSON naskah/glosarium):

A. Jalankan skrip diagnostik secara instan:
   * Klik ganda berkas: D:\pmn-framework\02_DIAGNOSTIK_DAN_AI_RESCUE.bat
   * Atau tekan opsi [9] di menu utama `00_PMN_WORKSPACE.bat`.
B. Skrip Python akan memindai seluruh direktori secara offline. Jika ada
   JSON yang rusak, skrip akan **menampilkan baris persisnya beserta kode yang salah**.
C. Sistem otomatis membuat berkas laporan khusus di folder utama:
   `D:\pmn-framework\LENGKAPI_DIAGNOSIS_UNTUK_AI.md`
D. Buka berkas tersebut, **salin (copy) seluruh isinya**, lalu tempelkan
   (*paste*) ke dalam chat AI web gratisan pilihan Anda (ChatGPT Free, Claude Web, dll.).
E. Laporan tersebut sudah dilengkapi instruksi sistem pembuka (*System Prompt*) 
   yang menceritakan arsitektur modular situs Anda secara lengkap. AI web tersebut 
   akan langsung memahami masalahnya dan memberikan solusi kode perbaikan secara tepat!

5. UTALITAS SNAPSHOT WORKSPACE (TOTAL SAFETY LOCK)
--------------------------------------------------------------------
Jika Anda khawatir modifikasi manual Anda merusak database glosarium, 
naskah bab modular, atau kode javascript visual:

A. Ambil Snapshot (Pencadangan Instan):
   * Klik ganda berkas: D:\pmn-framework\03_PULIHKAN_CADANGAN_SISTEM.bat (Opsi 1)
   * Atau tekan opsi [10] di menu utama (lalu pilih Opsi 1).
   * Sistem akan mengompresi seluruh file kerja Anda (`data/parts/*.json`, 
     `data/gl.json`, `index.ui.html`, `style.css`, `app.js`) menjadi satu berkas 
     ZIP super kecil di folder `backups/` dalam waktu kurang dari 0,5 detik.
B. Pulihkan / Rollback (Pengembalian Keadaan):
   * Jalankan menu yang sama, lalu pilih Opsi [2].
   * Sistem akan menampilkan seluruh daftar snapshot ZIP (diurutkan dari yang paling baru).
   * Masukkan angka indeks snapshot pilihan Anda.
   * **Safety Lock**: Sistem akan secara otomatis mengamankan berkas Anda saat ini 
     sebagai "auto_snapshot" terlebih dahulu (agar Anda tidak kehilangan pekerjaan), 
     lalu mengekstrak snapshot pilihan Anda, dan otomatis menjalankan compiler.
     Situs monolitik `index.html` Anda langsung pulih dan berfungsi kembali!
