1. ARSITEKTUR KARYA: HYBRID MODULAR-MONOLITH
--------------------------------------------------------------------
Untuk efisiensi token AI dan kemudahan kolaborasi, proyek ini dibagi:
*   KOTAK EDIT NASKAH (DATABASE): Folder `data/parts/` berisi 21 berkas
    JSON bab berukuran sangat kecil (~10KB hingga ~300KB).
*   KOTAK EDIT TAMPILAN (UI): File `index.ui.html` (~40KB).
*   KOTAK EDIT DESAIN & LOGIKA: Berkas `style.css` dan `app.js`.
*   HASIL AKHIR PRODUKSI: Berupa berkas HTML tunggal raksasa `index.html`
    (2.56 MB) yang dihasilkan otomatis oleh kompilator.

--------------------------------------------------------------------
2. PUSAT KENDALI UTAMA: 00_PMN_WORKSPACE.bat
--------------------------------------------------------------------
Komandan TIDAK PERLU lagi menjalankan banyak file bat terpisah atau
mengetik baris kode CMD. Cukup klik ganda (double-click) file sakti:
   D:\pmn-framework\00_PMN_WORKSPACE.bat

Menu konsol interaktif berwarna akan muncul. Tekan angka pilihan Komandan:
*   Tekan [1] : LIGHTNING COMPILE (Kompilasi Visual + Data + Corpus AI)
    --> Dilengkapi CHECKLIST PENGAMAN khusus untuk mencegah kompilasi kosong.
        Sistem akan menanyakan konfirmasi kesiapan berkas Word/UI baru Komandan,
        dan memverifikasi apakah database naskah sudah dipecah secara modular.
*   Tekan [2] : LAUNCH DEV SERVER (Server Preview Lokal)
    --> Menjalankan server HTTP lokal ringan pada http://localhost:8000
        dan otomatis membuka browser untuk meninjau perubahan secara realtime.
*   Tekan [3] : IMPORT WORD NASKAH (Impor Selektif & Kompilasi Otomatis)
    --> Fitur super canggih! Sistem otomatis memindai seluruh berkas Word
        (.docx) di folder docx_source/, menampilkan daftar berkas, dan memproses
        pilihan Komandan. Setelah terimpor dan terpecah secara modular, sistem
        akan otomatis men-compile website final secara instan.
*   Tekan [4] : SPLIT MONOLITH TO JSON (Pecah Database)
    --> Memotong paksa data/parts.json menjadi 21 file JSON modular kecil.
*   Tekan [5] : RESTORE STABLE BACKUP (Sistem Rollback / Batalkan Rilis)
    --> Fitur penyelamat darurat! Jika setelah Kompilasi Komandan mendeteksi ada
        kesalahan visual/isi, cukup pilih menu ini untuk menimpa website aktif
        dan mengembalikannya ke versi cadangan stabil (index.html.bak).
*   Tekan [6] : SYSTEM BLUEPRINT (Cetak Biru Arsitektur AI)
    --> Membuka ringkasan arsitektur file AI.md langsung di layar konsol.
*   Tekan [7] : SEARCH WORKSPACE (Pencarian Kode Cepat)
    --> Fitur pencarian kata kunci di seluruh berkas kode sumber lokal Anda.
*   Tekan [8] : SECURE META REMOVER (Pembersih Metadata & Backup Telegram)
    --> Meluncurkan pipa keamanan tingkat tinggi untuk membersihkan nama asli
        Anda dari dokumen draf (.docx/.pdf) secara lokal, mengaudit hasilnya,
        dan otomatis mengunggah cadangan sterilnya ke Telegram Log Group.

--------------------------------------------------------------------
3. CARA MENGUPDATE NASKAH DARI MICROSOFT WORD (.DOCX)
--------------------------------------------------------------------
Kini Komandan dapat mengimpor tulisan langsung dari file Word:
A. Taruh file Word manuskrip baru Komandan ke dalam folder:
   D:\pmn-framework\docx_source\
   (Contoh nama berkas: PMN_Framework_v116_2.docx).
B. Jalankan `00_PMN_WORKSPACE.bat` dan pilih opsi [3].
C. Sistem akan otomatis mendeteksi versi naskah, membaca isinya,
   menyinkronkan label versi ke `index.ui.html`, memecah bagian
   menjadi berkas JSON modular, dan memperbarui `index.html`.

*Catatan: Sistem secara otomatis membuat file cadangan aman dari
website stabil sebelumnya ke file index.html.bak.*

--------------------------------------------------------------------
4. CARA MENJALANKAN UJI KETAHANAN SKEMA (STRESSTEST)
--------------------------------------------------------------------
Jika Komandan ingin membuktikan secara ilmiah apakah sistem pembaca
otomatis mendeteksi bab baru dan menggambar tombol UI secara dinamis:
A. Jalankan perintah berikut di Git Bash / Terminal VS Code:

   menjalankan impor, memvalidasi keberhasilan kompilasi di HTML,

6. ALUR PANDUAN HARIAN: DARI NASKAH WORD HINGGA KE GITHUB (AWAM)
--------------------------------------------------------------------
Berikut adalah alur kerja harian yang sangat mudah dan aman bagi Anda untuk
memperbarui website naskah filosofi PMN dari nol hingga terbit di GitHub:

LANGKAH 1: EDIT NASKAH DI WORD
   1. Buka dokumen Word manuskrip Anda di Microsoft Word seperti biasa.
   2. Lakukan penulisan atau revisi bab sesuka hati Anda.
   3. Setelah selesai, lakukan 'Save As' dengan format nama versi baru, 
      contoh: `PMN_Framework_v117.docx` atau `PMN_Framework_v116_3.docx`.
   4. [PENTING!] Tutup Microsoft Word Anda agar file tidak terkunci!

LANGKAH 2: MASUKKAN KE FOLDER SUMBER
   1. Pindahkan file Word `.docx` baru Anda ke folder sumber:
      D:\pmn-framework\docx_source\
   2. Pastikan hanya ada satu file `.docx` aktif di folder tersebut.

LANGKAH 3: IMPOR & KOMPILASI DENGAN 00_PMN_WORKSPACE.bat
   1. Klik ganda file: D:\pmn-framework\00_PMN_WORKSPACE.bat
   2. Konsol interaktif PMN akan terbuka di layar Anda.
   3. Tekan tombol [3] lalu tekan Enter untuk mengimpor naskah Word Anda.
   4. Sistem otomatis membaca naskah Word, memecahnya menjadi file bab JSON,
      menyelaraskan label versi di web, dan mengompilasi file web final 
      index.html secara instan dalam 2 detik!

LANGKAH 4: STERILKAN METADATA (MENCEGAH KEBOCORAN IDENTITAS)
   1. Di konsol interaktif yang sama, tekan tombol [8] lalu tekan Enter.
   2. Ketik 'Y' saat ditawarkan opsi pembersihan massal.
   3. Skrip otomatis menyapu bersih nama Anda ("Ali Ikhsan") dari metadata 
      berkas draf, mengaudit hasilnya secara internal, dan mengunggah 
      cadangan steril tersebut ke grup Telegram Log Anda!
   4. [ALTERNATIF DRAG-AND-DROP]: Anda juga bisa langsung menyeret file 
      Word/PDF mentah Anda dan menjatuhkannya langsung di atas file batch:
      D:\pmn-framework\BERSIHKAN_DAN_BACKUP_DOKUMEN.bat

LANGKAH 5: PUSH KE GITHUB HANYA DENGAN SATU KLIK!
   1. Cari file launcher push otomatis di root folder Anda:
      D:\pmn-framework\KIRIM_KE_GITHUB.bat
   2. Klik ganda (double-click) file `KIRIM_KE_GITHUB.bat` tersebut.
   3. Masukkan pesan pembaruan singkat (misal: "Update Bab 6") lalu Enter.
   4. Biarkan skrip berjalan otomatis sampai selesai.
   5. [OK!] Perubahan website terbaru beserta draf steril Anda telah resmi
      dan aman terbit di GitHub publik tanpa kebocoran data pribadi!
