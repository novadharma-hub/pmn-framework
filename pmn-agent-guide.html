Using PMN Agent
without losing the manuscript

A grounded workflow for ChatGPT, Gemini, and future backend-backed PMN Agent modes.

Open homepage Open orientation Open compressed core

PMN Agent is most useful when it is grounded in the manuscript rather than asked to "figure out the website" on its own. The current PMN site is a static reader: section anchors such as #4.4 and #15.15 are navigation states inside one page, not separate documents that an external model can reliably browse, scroll, and interpret like a human reader.

So the right workflow is retrieval first, model second. Let the PMN site gather the relevant sections, glossary terms, and active context. Then send that package into ChatGPT, Gemini, or a future secure backend agent. This guide is written for that architecture.

Step 01

Choose the right PMN workflow

Not every use-case needs the same setup. If you just want a grounded answer quickly, the built-in handoff from the PMN site is now the best default. If you want the model to work from the whole manuscript in one conversation, upload the document or give it a larger context pack.

Grounded handoff
Use the homepage or reader page. PMN retrieves relevant sections and glossary terms, then opens ChatGPT or Gemini with a prepared prompt.
Recommended
Full document upload
Useful for long sessions where you want one model to stay inside PMN for a while. Best when the model can hold long context reliably.

Paste the URL only
Do not rely on this. Static readers and hash anchors are not enough to guarantee that the model actually read the relevant PMN material.

Current site reality The public PMN site can already build grounded prompts from the manuscript data embedded in the page. What it cannot safely do yet is call commercial models inline from the browser without a secure backend.
Step 02

Know what the model should read

The model should not be asked to infer PMN from vague memory, ideology labels, or generic political-philosophy priors. It should read from a PMN context pack containing the actual section text, section IDs, and glossary terms most relevant to your question.

Important: GitHub Pages, single-page readers, and hash anchors do not guarantee reliable external browsing. If your question is really about 4.4, 7.3, or 15.15, include those anchors in the prompt or open them through the PMN site so the handoff can package the right text.

The safest ordering is:

1. Identify the sectionName the section, case, or problem you care about.
2. Retrieve the contextLet the site gather the most relevant PMN passages and glossary terms.
3. Ask for analysisOnly then ask the model to diagnose, compare, or stress-test.
4. Demand citationsAsk the model to point back to section IDs, not just smooth prose.
Step 03

Prime the model as PMN Agent

Once you have either the manuscript upload or a PMN context pack ready, use a priming prompt that makes the model operate from within the framework instead of merely summarizing it.

Priming prompt
You are acting as PMN Agent for Progressive Materialist Naturalism (PMN) v97 by Nova Dharma. Use the supplied PMN context pack as your primary source for this conversation. Do not treat PMN as a generic ideology label. Treat it as a structured framework with section anchors, glossary terms, and internal constraints. Your task is not to summarize the text unless I ask. Your task is to reason through my question using PMN's architecture: - epistemology and revision discipline - ontological commitments - the biological floor and evaluative criterion - structural variables, institutional analysis, and failure modes - the compressed core in 15.15 when simplification is needed When you answer: 1. stay grounded in the supplied PMN material 2. name the most relevant section IDs when possible 3. distinguish confident claims from evidential gaps 4. prefer diagnosis over vague rhetoric Start by confirming, in under 150 words, what PMN treats as its evaluative floor and why that floor does not exhaust the full horizon of becoming.

A good confirmation will mention that PMN starts from the biological reality of suffering as a non-arbitrary evaluative floor, while still insisting that the ceiling of becoming is richer than mere pain-reduction.

Step 04

Choose the operational role

After the priming step, give the model a role that matches your task. The sharper the role, the less likely the model is to drift into generic commentary.

General analyst prompt
From this point forward, operate as PMN Agent. Reason from within the framework rather than describing it from outside. Use PMN's evaluative criterion, structural variables, and failure mode discipline as your own analytic starting points unless I explicitly ask you to step outside them. When I bring a case: - identify the structural arrangement being evaluated - name which variables or sections matter most - say what PMN would need to see before reaching confidence - cite section IDs where doing so will improve precision
Diagnostic mode prompt
Operate as a PMN diagnostic analyst. For each case I bring: - identify the dominant structural variables - name the most activated PMN questions or tensions - identify the likely failure modes in the analysis itself - separate what is evidenced from what is only plausible - end with the key missing evidence needed for stronger confidence You are not required to reach a verdict if PMN would treat the evidence as incomplete.
Debate partner prompt
Operate as a PMN debate partner. For each argument I give you: 1. reconstruct the strongest version of it 2. say what PMN would find structurally persuasive about it 3. identify where the argument becomes architecturally incomplete 4. test whether the same lens is being applied consistently The goal is not rhetorical victory. The goal is analytical honesty and pressure-testing.
Step 05

Ask questions that pull real PMN analysis out of the model

Weak prompts ask whether PMN is "for" or "against" something. Strong prompts name the arrangement, the claimed justification, and the section or variable that should govern the analysis.

User
Using PMN, analyze whether a platform-labor arrangement can be defended by worker "preference" when the wider choice set is structurally degraded. Anchor your answer in the relevant sections and tell me what evidence would change the assessment.
PMN Agent

This is a stronger PMN question because it does not ask for a vibes-level judgment. It identifies the contested justification, names the structural issue, and invites an evidence-sensitive answer.

A grounded response should bring in the architecture around structural suffering, institutional betrayal, visibility, and the problem of preferences formed inside constrained choice sets. It should also state what evidence would weaken the criticism: real exit options, durable protections, and materially different bargaining conditions.

User
Read 15.15 as the compressed core, then explain PMN to a new reader in plain language without flattening the difference between the minimum anchor and the horizon of becoming.
PMN Agent

This is high-yield because it tells the model exactly which PMN compression layer to use and which conceptual distinction must survive simplification.

Without that instruction, many models collapse PMN into "reduce suffering" only. The prompt prevents that flattening by explicitly protecting the larger developmental horizon.

Better prompt pattern Use this template: "Using PMN, analyze arrangement or case. Name the most relevant section IDs, say which structural variables matter most, and identify what evidence would most likely change the conclusion."
Step 06

Manage drift, overconfidence, and false smoothness

Even strong models drift. Over time they may revert to generic political language, compress distinctions they should preserve, or answer too fluently for the evidence actually available.

Reset prompt
Return to PMN mode. Re-ground the answer in the supplied PMN material, name the most relevant section IDs, identify the structural variables at stake, and say what the main evidential gap is before acting more confident than the evidence allows. If you are simplifying, do not collapse the difference between the minimum anchor and the fuller horizon of becoming.
Ask for citationsIf the model cannot point back to PMN sections, it is probably freelancing.
Ask what would falsify itThis is the fastest way to detect fake confidence.
Check the missing evidenceA PMN answer should tell you what it still does not know.
Compare against the readerOpen the cited sections and see whether the answer preserved the actual PMN distinction.
Static build limitation: the public site can prepare excellent PMN context packs, but it is not yet the final in-page agent architecture. Secure inline answers need a backend retrieval layer. Until then, the best public workflow is grounded handoff, not pretending the model read the URL by itself.
Step 07

For builders: what the PMN Agent should eventually become

If you are improving the PMN site itself, the correct direction is to move from browser-only prompting toward backend retrieval. The model should read the PMN database, not the URL.

Store sectionsEach section should be queryable by ID, title, body, and relations.
Store glossary termsTerms should link back to the sections that define and use them.
Build context packs server-sideRetrieve relevant passages before the model answers.
Return citationsEvery answer should be able to point back to section IDs that justify it.

That is how PMN Agent stops being a prompt launcher and becomes a real framework-specific assistant.

PMN should make the model more disciplined, not merely more confident. If the answer becomes smoother while the manuscript disappears, the workflow has failed. Keep the text in view, keep the section IDs alive, and make the model earn its certainty.
